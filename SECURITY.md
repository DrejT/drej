# Security Policy

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Send a description of the issue to **vivekpatel4049@gmail.com**. Include:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept if possible
- Any suggested fixes

You'll receive a response within 72 hours. We'll work with you to understand and address the issue before any public disclosure.

## Supported versions

We apply security fixes to the latest published version only.
