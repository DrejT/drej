---
title: drejx kill
description: Stop a sandbox by ID.
---

```bash
bunx drejx kill <sandbox-id>
```

## Arguments

| Argument     | Description                                                    |
| ------------ | ---------------------------------------------------------------------------------- |
| `sandbox-id` | The sandbox ID, as shown by `drejx agents` or printed by `drejx spawn`/`drejx fork`. Required. |

## What it does

Connects to the sandbox via a live control-plane check (the same authoritative liveness check `drejx prompt` uses — see [Why sandbox ID, not name](/docs/drejx/commands/prompt#why-sandbox-id-not-name)) and closes it, deleting the container and releasing its resources.

## Example

```bash
bunx drejx kill 4af65c3b-24a2-4fd1-999d-918faa9b97fd
```

```
Killed sandbox 4af65c3b-24a2-4fd1-999d-918faa9b97fd
```

## Notes

- `kill` only stops the sandbox it's given — it does not recursively stop any children that sandbox may have forked via `drejx fork`. Kill each one individually, or check `drejx agents` for the full list including untracked (agent-spawned) sandboxes.
- If the sandbox is already stopped or never existed, the underlying live check fails and `kill` errors out rather than silently succeeding.
